# Setup Guide

## Overview

Three components need to be running:

```
NapCatQQ (Docker)  ←  QQ client, provides OneBot v11 API
    ↕ HTTP + WebSocket
Amadeus-QQ-MCP (uv)  ←  MCP bridge, buffers messages
    ↕ streamable-http (HTTP)
agent-cli (tsx)  ←  Social agent, connects to LLM
```

## Prerequisites

- **Docker** (Docker Desktop recommended for WSL2 integration)
- **Python 3.10+** with `uv` package manager
- **Node.js 18+** with npm

## Step 1: NapCatQQ

```bash
# Clone Amadeus (which includes the NapCat docker-compose)
git clone https://github.com/JulesLiu390/Amadeus-QQ-MCP.git ~/projects/Amadeus-QQ-MCP
cd ~/projects/Amadeus-QQ-MCP

# Start NapCat
docker compose up -d napcat

# Check login status
docker logs napcat
# → Scan QR code if this is first login
# → Wait for "[NapCat] 登录成功"
```

### OneBot Configuration

After first login, NapCat creates config files in `napcat/config/`. Enable HTTP and WebSocket:

```bash
# Make config writable (first time only)
sudo chown -R $USER:$USER napcat/config/

# Edit napcat/config/onebot11_{your_qq}.json
```

Required configuration:

```json
{
  "network": {
    "httpServers": [{
      "name": "http_main",
      "enable": true,
      "port": 3000,
      "host": "0.0.0.0",
      "enableCors": true,
      "heartbeatInterval": 30000,
      "messagePostFormat": "array",
      "reportSelfMessage": false,
      "token": ""
    }],
    "websocketServers": [{
      "name": "ws_main",
      "enable": true,
      "host": "0.0.0.0",
      "port": 3001,
      "heartbeatInterval": 30000,
      "messagePostFormat": "array",
      "reportSelfMessage": false,
      "token": ""
    }]
  }
}
```

```bash
# Restart to apply config
docker compose restart napcat
```

Verify:

```bash
curl -X POST http://127.0.0.1:3000/get_login_info
# → {"status":"ok","retcode":0,"data":{...},"message":"NapCat4 Is Running"}
```

## Step 2: Amadeus-QQ-MCP

```bash
cd ~/projects/Amadeus-QQ-MCP

# Install dependencies
uv sync

# Install Playwright (for screenshot_chat tool)
uv run playwright install chromium

# Test standalone (MCP server will start, then exit after 10s)
timeout 10 uv run qq-agent-mcp --qq YOUR_QQ_NUMBER
# → "Starting MCP Server (QQ: ..., OneBot: http://127.0.0.1:3000)"
```

Amadeus is run as a standalone MCP server via streamable-http transport.
agent-cli connects to it over HTTP.

```bash
# Terminal 1: Start Amadeus MCP
cd ~/projects/Amadeus-QQ-MCP
uv run qq-agent-mcp --qq YOUR_QQ --transport streamable-http --host 127.0.0.1 --port 8099 --groups YOUR_GROUP_ID --friends YOUR_FRIEND_ID
```

## Step 3: agent-cli

```bash
cd ~/projects/agent-cli

# Install dependencies
npm install

# Initialize social directory
npm start -- init

# Configure .env
cp .env.example .env
# Edit .env with:
#   - LLM_API_KEY (required)
#   - LLM_BASE_URL (recommended: https://opencode.ai/zen/go/v1)
#   - LLM_MODEL (e.g., deepseek-v4-flash)
#   - OWNER_QQ (your QQ number)
#   - WATCHED_GROUPS (comma-separated)
#   - WATCHED_FRIENDS (comma-separated)

# Verify LLM connectivity
npm run test-llm

# Start the agent
npm start
```

## Step 4: Configuration

### `.env` (environment variables)

```ini
# LLM Provider
LLM_API_KEY=sk-your-key
LLM_BASE_URL=https://opencode.ai/zen/go/v1
LLM_MODEL=deepseek-v4-flash

# Owner QQ (for security token identification)
OWNER_QQ=123456789

# Watched targets
WATCHED_GROUPS=699242647
WATCHED_FRIENDS=2532452182

# Context limit (default 64000, max for model)
MAX_CONTEXT_TOKENS=200000
```

### `social/config.json` (persistent config)

Generated by `npm start -- init`. Edit directly or use the config-set command:

```bash
npm start -- config-set llm.model deepseek-v4-flash
```

## Step 5: Full Startup Sequence

You need 4 components running. Start them in order:

```bash
# === Terminal 1: NapCat (QQ client) ===
cd ~/projects/Amadeus-QQ-MCP
docker compose up -d napcat

# Verify NapCat is logged in
curl -X POST http://127.0.0.1:3000/get_login_info
# → {"status":"ok","retcode":0,"data":{...}}

# === Terminal 1 (or background): SearXNG (optional, web search) ===
cd ~/projects/searxng-local
docker compose up -d

# Verify SearXNG
curl "http://127.0.0.1:8080/search?q=test&format=json"

# === Terminal 2: Amadeus MCP (bridge) ===
cd ~/projects/Amadeus-QQ-MCP
uv run qq-agent-mcp --qq YOUR_QQ --transport streamable-http \
  --host 127.0.0.1 --port 8099 \
  --groups YOUR_GROUP_ID --friends YOUR_FRIEND_ID

# === Terminal 3: agent-cli ===
cd ~/projects/agent-cli
npm start
```

### Quick Restart (daily use)

If all Docker services are already running, just restart Amadeus + agent:

```bash
# Terminal 1: restart Amadeus
cd ~/projects/Amadeus-QQ-MCP
uv run qq-agent-mcp --qq YOUR_QQ --transport streamable-http \
  --host 127.0.0.1 --port 8099 \
  --groups YOUR_GROUP_ID --friends YOUR_FRIEND_ID

# Terminal 2: restart agent
cd ~/projects/agent-cli
npm start
```

NapCat and SearXNG use `restart: always` so they auto-start with Docker.

## Step 6: Local Web Search (SearXNG)

The agent can search the web in real-time using a local SearXNG instance.

### Deployment

```bash
# Create deployment directory
mkdir -p ~/projects/searxng-local/core-config

# Create docker-compose.yml and .env
# (Full files at ~/projects/searxng-local/)
```

Minimal `docker-compose.yml`:

```yaml
name: searxng

services:
  core:
    container_name: searxng-core
    image: docker.io/searxng/searxng:latest
    restart: always
    ports:
      - 127.0.0.1:8080:8080
    environment:
      FORCE_OWNERSHIP: "true"
    volumes:
      - ./core-config:/etc/searxng
      - core-data:/var/cache/searxng

  valkey:
    container_name: searxng-valkey
    image: docker.io/valkey/valkey:9-alpine
    command: valkey-server --save 30 1 --loglevel warning
    restart: always
    volumes:
      - valkey-data:/data

volumes:
  core-data:
  valkey-data:
```

Minimal `core-config/settings.yml` (enable JSON API for the agent):

```yaml
use_default_settings:
  engines:
    keep_only:
      - google
      - brave
      - bing
      - news
      - wikipedia
      - duckduckgo

general:
  debug: false

search:
  formats:
    - html
    - json

server:
  secret_key: "<generate a random string>"
  bind_address: "[::]"
  public_instance: false

outgoing:
  request_timeout: 15.0
  max_request_timeout: 30.0
```

Start it alongside NapCat (see Step 5). Verify:

```bash
curl "http://127.0.0.1:8080/search?q=test&format=json"
```

Enable web search in your agent config (`social/config.json`):

```json
{
  "webSearch": {
    "enabled": true,
    "baseUrl": "http://127.0.0.1:8080",
    "timeoutMs": 10000,
    "maxResults": 5
  }
}
```

### opencode MCP Configuration (Optional)

If you also want opencode to use the same search:

```json
{
  "mcp": {
    "searxng": {
      "type": "local",
      "command": ["npx", "-y", "mcp-searxng"],
      "enabled": true,
      "env": {
        "SEARXNG_URL": "http://127.0.0.1:8080"
      }
    }
  }
}
```

Add to `~/.config/opencode/opencode.json` (global) or `./opencode.json` (project), then restart opencode.

```bash
cd ~/projects/agent-cli

```

This uses a built-in mock MCP server that simulates QQ group messages.

## Troubleshooting

### LLM calls fail with "fetch failed"
- Check your internet connection and proxy settings
- OpenCode Go API works through most proxies; DeepSeek official may SSL timeout
- Try `npm run test-llm` to isolate LLM issues

### No new messages trigger eval
- Verify NapCat is receiving messages: `docker logs napcat | grep "接收"`
- Verify the WebSocket between Amadeus and NapCat: messages appear in `batch_get_recent_context`
- The debounce timer waits 3s before evaluating; send multiple messages to test

### `decide({})` with empty args
- This means the LLM called decide without parameters. The `inputSchema` should prevent this.
- Check that you're using the latest version of the code (tool definitions updated)
